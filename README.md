# 多主题签到系统（全栈版 v2）

一个支持**多活动 / 多主题**的会议签到工具。每个活动有独立的「主题编码」，参与者通过带编码的链接进入对应签到页；**管理员**登录后自行创建并管理多个会议（主题），配置报名题目与名单校验；**开发者（超级管理员）**通过独立控制台创建管理员、管理全部主题与数据。

> 密码不再写死在代码里，全部通过界面配置并加密存储。管理员登录**只需账号 + 密码**，登录后即可创建并管理自己的多个会议。

---

## 一、核心特性

- 🏷️ **主题编码隔离**：每个活动一个唯一编码（不可重复），数据互相独立
- 🔗 **链接变式**：参与者访问 `?code=主题编码` 即进入对应活动页（名称、配色自动加载）
- 📝 **报名题目（填空 / 选择题）**：每个主题可自定义报名时参与者需作答的题目，支持必填、选择题选项，答案随签到记录保存并在后台/导出中可见
- ✅ **报名名单校验**：可上传名单并开启「名单校验」——仅名单内人员可签到、自动带出姓名单位；支持「严格模式」（名单外禁止报名/签到）或「允许非名单人员自主报名」
- 👤 **管理员体系**：可新增多名管理员，密码加密存储（PBKDF2）；**管理员登录仅需账号 + 密码**
- 🏢 **一个管理员可创建多个会议**：登录后在「我的会议」里自由创建、管理、删除多个主题，互不干扰
- 🔐 **权限隔离**：管理员仅能管理自己创建的会议；未授权无法访问他人数据
- 🛠️ **开发者控制台**：超级管理员可创建管理员、修改密码、创建/配置任意主题、查看**所有主题**数据
- 📱 **会议签到码（二维码）**：创建/配置会议后一键生成签到二维码（PNG），可下载打印或复制链接分享
- 📤 **数据导出**：管理员可导出本主题 CSV（含题目答案），开发者可导出全部 CSV
- 📱 **多设备实时共享**：数据集中存储于服务器

## 三、本地运行

需要本机已安装 Node.js（建议 18+）。

```bash
cd checkin-multitenant
npm install
node server.js
```

启动后访问：
- 参与者签到页：http://localhost:3000/?code=会议2026
- 管理后台：http://localhost:3000/admin
- 开发者控制台：http://localhost:3000/console

> 首次使用必须先访问 `/console` 创建超级管理员（开发者）账号；普通管理员由超级管理员在控制台创建。

## 四、部署到云端（Render）

详细见 **`部署指南.md`**。要点：

1. 将 `checkin-multitenant/` 目录上传到 GitHub 仓库
2. 在 [Render](https://render.com) 选择 **New → Blueprint**，连接仓库（自动读取 `render.yaml`，已通过 `rootDir` 指向本目录）
3. 部署完成后：
   - 访问 `https://你的地址/console` 创建超级管理员
   - 在控制台创建主题（得到主题编码）
   - 把 `https://你的地址/?code=主题编码` 生成二维码分享给参与者

## 五、使用流程

### 第 1 步：初始化（仅一次）
部署后访问 `/console` → 创建超级管理员（开发者）账号。该账号即「开发者」，可查看所有主题数据。

### 第 2 步：创建管理员（开发者）
在 `/console` 控制台「管理员」标签：
- **新增管理员**：创建普通管理员账号（供各活动负责人使用），只需用户名 + 密码
- 普通管理员登录后将**自行创建并管理自己的多个会议**；开发者可管理全部

### 第 3 步：管理员创建会议（主题）
普通管理员访问 `/admin` → 用账号密码登录 → 在「我的会议」点击「创建新会议」：
- 填写「主题编码」（唯一，参与者链接用）+「会议名称」+「主题色」
- 一个管理员可创建任意多个会议，互不干扰
- 也可由开发者在 `/console` 的「主题管理」中创建

### 第 4 步：分享签到链接
在「我的会议」里点对应会议的「管理」，页面顶部会显示参与者签到链接 `https://你的地址/?code=主题编码`，生成二维码分享即可。

### 第 5 步：配置报名题目与名单校验（管理员 / 开发者）
在 `/admin` 进入某会议「管理」页，在「报名设置」区：
- **报名题目**：点击「添加题目」，选择「填空」或「选择题」，填写题面，选择题可填多个选项，可设「必填」。保存后参与者报名/签到时必须作答。
- **名单校验**：勾选「启用报名名单校验」→ 仅名单内人员可签到，且自动带出姓名单位。再视需要勾选「允许非名单人员自主报名」：
  - 勾选（默认）：名单外人员仍可自助报名，并在记录中标注为「非名单」
  - 不勾选（严格模式）：名单外手机号无法报名或签到
- 题目答案会随签到记录保存，在后台表格与导出 CSV 中可见
- 开发者也可在 `/console` 的「主题管理 → 配置」中设置任意主题的题目与校验

### 第 6 步：参与者签到
参与者输入手机号 → 系统识别状态：
- 在名单中 → 自动带出姓名单位，补全信息 + 作答题目 → 签到
- 新参与者 → 填写姓名单位 + 作答题目 → 报名并签到（严格模式下名单外会提示无法报名）
- 多次签到 → 提示已签到

### 第 7 步：管理员查看与管理
访问 `/admin`，用「管理员账号 + 密码」登录后，在「我的会议」选择某会议进入「管理」，可：
- 查看报名 / 签到统计与明细（含题目答案）
- 批量导入名单（姓名,手机号,单位）
- 手动补录签到
- 导出 CSV（含题目答案列）
- 删除本会议（含全部数据）

### 第 8 步：开发者总后台
开发者在 `/console` 登录后，切换「全部数据」标签页，可查看并导出**所有主题**的完整数据。

## 六、API 接口

| 接口 | 方法 | 说明 | 鉴权 |
|------|------|------|------|
| `/api/health` | GET | 健康检查 | 无 |
| `/api/theme?code=` | GET | 主题公开信息 | 无 |
| `/api/lookup` | POST | 查手机号状态 | 需 code |
| `/api/checkin` | POST | 名单内签到 | 需 code |
| `/api/signup` | POST | 报名并签到 | 需 code |
| `/api/admin/login` | POST | 管理员登录（账号+密码） | — |
| `/api/admin/themes` | POST | 管理员可管理的会议列表 | Token |
| `/api/admin/theme/create` | POST | 管理员创建会议（主题） | Token |
| `/api/admin/theme/config` | POST | 配置某会议题目/校验策略 | Token |
| `/api/admin/theme/delete` | POST | 删除某会议 | Token |
| `/api/admin/data` | POST | 指定会议数据（body 带 code） | Token |
| `/api/admin/import` | POST | 导入名单 | Token |
| `/api/admin/checkin` | POST | 手动签到 | Token |
| `/api/admin/export` | POST | 导出 CSV | Token |
| `/api/admin/reset` | POST | 重置本会议 | Token |
| `/api/console/setup` | POST | 创建首个超级管理员 | 仅首次 |
| `/api/console/login` | POST | 超级管理员登录 | — |
| `/api/console/admin/create` | POST | 新增管理员 | 超级 Token |
| `/api/console/admin/password` | POST | 修改密码 | 超级 Token |
| `/api/console/theme/create` | POST | 创建主题（编码唯一） | 超级 Token |
| `/api/console/theme/config` | POST | 配置任意主题题目/校验 | 超级 Token |
| `/api/console/data/all` | POST | 全部主题数据 | 超级 Token |

## 七、常见问题

**Q：忘记超级管理员密码怎么办？**
A：登录服务器，删除 `data.json` 后重启，重新访问 `/console` 初始化（会清空所有数据，请先备份）。

**Q：主题编码能重复吗？**
A：不能。创建主题时系统会校验唯一性，重复会报错。

**Q：普通管理员能看到其他主题的数据吗？**
A：不能。普通管理员仅能管理自己创建的主题；只有超级管理员（开发者）可在控制台查看全部。

**Q：数据存在哪里？会被重置吗？**
A：数据存于云端 `data.json`（临时磁盘），重新部署会被重置，请定期用「导出 CSV」备份。
